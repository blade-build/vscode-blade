# BUILD 语言特性

BUILD（以及 `BLADE_ROOT`）文件被识别为 `blade-build` 语言。

## 语法高亮

TextMate 语法对注释、字符串、数字、布尔/`None` 常量、已知规则调用
（`cc_library`、`proto_library`……）以及常见属性名（`name`、`srcs`、`deps`……）
进行高亮。

## 大纲

**大纲** 视图与面包屑会按 `name` 列出当前 BUILD 文件中声明的目标。测试、可执行和
库规则使用不同的符号类型。

## 跳转到定义

将光标置于依赖标签上并执行 **跳转到定义**（F12），即可跳到该目标的声明：

- 绝对标签 —— `//common/base:string`；
- 相对标签 —— `:helper`（同一个包）。

将打开目标的 BUILD 文件并把光标定位到其 `name`。

## 补全

- 语句开头的 **规则类型**，以带 `name` 占位符的代码片段展开。
- 规则调用内部的 **属性名**。
- `deps` / `testdata` 列表内部的 **依赖标签** —— 提供工作区内的所有目标，并标注其
  规则类型。

## 悬停

悬停在依赖标签上会显示该目标的类型及其源文件 / 依赖数量。

## 诊断

无法解析到任何已知目标的绝对标签（`//path:name`）会被标记为告警。相对标签以及
系统 / 第三方标签不会被标记，以避免误报。当目标模型为空或处于错误状态时诊断会被
抑制，因此绝不会在过时视图上触发。
